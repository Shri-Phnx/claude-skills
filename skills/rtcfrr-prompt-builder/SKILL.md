# RTCFRR Prompt Builder
### R·T·C·F·R·R — Universal Prompt Template Skill

## What this skill does

Generates a complete, structured AI prompt using the six-section R·T·C·F·R·R framework. Each section serves a distinct purpose, and together they eliminate vague outputs, AI drift, and generic results.

---

## The Six Sections

### R — ROLE
Define the expert persona Claude should inhabit.
```
ROLE: You are a [job title / expert persona] with [X years] of experience in [domain],
specialising in [focus area] for [market/audience].
```
**Why it matters:** Sets the knowledge frame, tone authority, and perspective filter for the entire response.

---

### T — TASK
One clear action + one specific deliverable.
```
TASK: [Action verb] + [specific deliverable] for [platform / channel / purpose].
```
**Why it matters:** Prevents Claude from broadening or narrowing scope on its own.

---

### C — CONTEXT
Five sub-fields that ground the output in reality.
```
CONTEXT:
- Audience:         [who will read/use/act on this]
- Problem:          [pain point or gap being addressed]
- Goal:             [what success looks like]
- My background:    [relevant credential or data point to weave in]
- Additional context: [anything else the AI must know]
```
**Why it matters:** The difference between generic and hyper-relevant output lives here.

---

### F — FORMAT
Controls the shape and presentation of the output.
```
FORMAT:
- Structure:    [sections or flow — e.g. Hook → Problem → Insight → CTA]
- Length:       [word/line/page limit]
- Language:     [tone] + [British / US English]
- Output type:  [bullets / prose / table / JSON / numbered list / code]
- Extras:       [hashtags / headers / citations — include or exclude]
```
**Why it matters:** Prevents Claude from defaulting to its preferred layout rather than yours.

---

### R — RULES + REFERENCES
Non-negotiables that Claude must follow or avoid.
```
RULES + REFERENCES:
Do:
- [Non-negotiable to include]
- [Style or tone requirement]

Do not:
- [Hard restriction — phrase, format, or approach to avoid]
- [Common AI drift to block]

Reference example (what good looks like):
"[Paste a sample line or structure you want matched]"
```
**Why it matters:** Blocks the five most common failure modes: wrong tone, wrong format, wrong scope, unwanted phrases, and AI-sounding language.

---

### R — REVIEW + REFINE
A built-in self-audit before the final output.
```
REVIEW + REFINE:
After generating, audit against these criteria:
- [Criterion 1]
- [Criterion 2]
- [Criterion 3]
- [Criterion 4]
Then rewrite the final version only. Do not show the draft.
```
**Why it matters:** Moves quality control inside the prompt itself — no need for a follow-up correction round.

## How to use this skill

### Step 1 — Identify the use case
Ask (or infer from context) what the user needs the prompt for. Common use cases:
- CV / resume tailoring for a specific JD
- LinkedIn post or thought leadership content
- Recruiter outreach message (connection request or InMail)
- CareerForge coaching content (Instagram Reels, LinkedIn carousel)
- Job application cover letter or email
- Technical documentation or SOW
- Any other professional deliverable

### Step 2 — Gather the six inputs
For each section, either extract from context already provided or ask the user directly. Minimum required:
- **ROLE**: Expert persona + domain
- **TASK**: Action verb + deliverable
- **CONTEXT**: Audience and goal (others can be inferred)
- **FORMAT**: Output type and length
- **RULES**: At least one Do and one Do Not
- **REVIEW**: At least two audit criteria

If the user provides partial information, fill sensible defaults and flag assumptions.

### Step 3 — Assemble and execute the prompt
Build the full R·T·C·F·R·R prompt block, then execute it immediately in the same response (unless the user only wants the prompt template itself).

Structure the output as:
1. The filled R·T·C·F·R·R prompt (in a code block or clearly labelled section)
2. The output generated by running that prompt

### Step 4 — Offer to refine
After delivering, ask: *"Would you like to adjust any section — Role, Format, Rules, or the audit criteria?"*

## Failure modes to avoid

| Failure | Cause | Fix |
|---|---|---|
| Output is generic | CONTEXT is too thin | Add Audience + Problem + My background |
| Wrong tone | No FORMAT language field | Specify tone + British/US English explicitly |
| Too long / too short | No length limit | Add word/line count to FORMAT |
| AI-sounding phrases leak in | No Rules block | Add "Do not use AI-sounding openers" to Do Not |
| Draft shown before final | No REVIEW section | Always include the "rewrite final version only" instruction |
| Wrong expertise angle | Vague ROLE | Specify years, domain, AND focus area |
